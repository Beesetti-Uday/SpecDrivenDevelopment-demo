## Why

Users need a single coherent terminal experience that combines task management, recurring planning, journaling, and follow-up workflows. This change solves the fragmentation of separate interfaces by unifying these capabilities under one terminal UI.

## What Changes

- Introduce a unified terminal interface that surfaces todos, recurring schedules, journal entries, and follow-up tasks together.
- Add a consolidated command flow for reviewing daily work, capturing journal reflections, and managing follow-up actions.
- Improve navigation and visibility so users can move seamlessly between tasks, recurring items, journals, and follow-ups.

## Capabilities

### New Capabilities
- `terminal-ui`: Unified terminal user interface for managing todos, journals, recurring schedules, and follow-up actions.
- `workflow-integration`: Integrated workflow state that links tasks, recurring schedule items, journal entries, and follow-up actions into a cohesive experience.
- `review-dashboard`: Terminal dashboard for reviewing current work, upcoming recurring tasks, recent journal entries, and pending follow-ups.

### Modified Capabilities
- None: This change adds new integration and UI behavior without altering existing core requirements.

## Impact

- Adds new terminal UI flows and commands to the existing terminal app.
- Updates command structure and user experience for task, journal, recurring, and follow-up workflows.
- May require new data relationships between tasks, journal entries, and follow-up metadata.
